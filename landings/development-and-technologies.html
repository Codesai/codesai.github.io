.Net y Javascript

Prolongada experiencia con C#

Herramientas Microsoft y Open Source

Ecosistema JavaScript

En Codesai somos expertos en desarrollo con tecnologías .Net. Sobre todo con C#, en el cual hemos trabajando desde la versión 1.1 del framework hasta hoy en día en la versión 6. Durante estos años hemos trabajado tanto con herramientas Microsoft como con Open Source como por ejemplo CastleProject (usamos Dynamic Proxy para implementar AOP) y el proyecto Mono. También estamos usando F# para algunas partes de las aplicaciones y para algunos tipos de tests.

Frontend

Frameworks de vista como Knockout y Angular

Interfaz web con HTML5, CSS3, jQuery

Aplicaciones de escritorio para Windows

Hemos hecho diversas aplicaciones con Windows.Forms, con WPF y también para WinRT (Windows 8 y posteriores).

Ya en el lado frontend, somos unos apasionados de JavaScript. Una de nuestras librerías favoritas es Knockout porque se integra muy bien con todo el ecosistema y no es tan intrusiva y prescriptiva como algunos frameworks. Si bien es cierto que a veces simplemente escribimos vanilla JavaScript y recurrimos a algo de jQuery. Otras incluso directamente accedemos al DOM con la API de HTML 5.

También tenemos experiencia con React y con Angular. Hemos desarrollado aplicaciones con ES6 usando Babel para la compilación y últimamente estamos probando TypeScript como una capa adicional para tipar algunos objetos.

Backend

Arquitectura Hexagonal

Domain Driven Design

ASP.Net MVC, Dapper

Típicamente el backend de las aplicaciones lo desarrollamos con ASP.Net MVC.

Así para algunos proyectos accedemos a datos mediante herramientas sencillas como Dapper y para otros podemos escoger EntityFramework si hace falta. Antes de que existiera EntityFramework usabamos ActiveRecord de CastleProject o directamente NHibernate. En los proyectos más recientes estamos encantados con la simplicidad que nos aporta Dapper (de los creadores de StackOverflow).

7

Nuestra arquitectura más usada es la Hexagonal, que encaja perfectamente con DDD (Domain Driven Design) lo cual nos permite separar capas y tecnología para que las librerías de acceso a base de datos o de vista no contamine el core de negocio.

Entrega de valor

Desarrollo ágil a medida

Entregas en ciclos cortos cada 15 días

Flexibles, transparentes y cercanos

Hacemos desarrollo ágil a medida. Consideramos al cliente como parte del equipo de desarrollo, haciendo juntos el análisis del problema y evaluando juntos posibles soluciones.

Entregamos software en ciclos cortos de quince días. Es decir, cada quince días cumplimos con un hito que aporta valor al negocio. Semanalmente hay entregas parciales y demos de los avances que se están realizando. Partimos el proyecto en distintas releases y ayudamos al cliente a priorizar cuáles son las que deben implementarse primero. El cliente es dueño del código fuente en todo momento así como de la documentación técnica que vamos generando con cada entrega.

En el arranque de los proyectos nos desplazamos a donde está el cliente y los usuarios para conocer en primera persona cuál es su contexto y sus necesidades y para hacer equipo. Dedicamos varias jornadas de intenso trabajo de descubrimiento y definición del problema. Nos apoyamos en diversas técnicas de análisis como pueda ser Design Thinking o BDD (Behavior Driven Development).

Nuestra máxima es la colaboración con el cliente y la entrega de valor constante.

Somos flexibles, los requisitos pueden cambiar tanto como el cliente lo necesite. Cada dos semanas hay oportunidad de definir cuáles son los detalles del siguiente hito a desarrollar. Qué hay que cambiar o añadir. Somos transparentes, el cliente siempre puede hablar con cualquier miembro del equipo de desarrollo.

Hacemos reuniones de seguimiento diarias de diez minutos a las que el cliente está invitado, además de tener acceso directo al sistema de control de versiones, que típicamente está en la nube.

Nuestra oferta es la de software de calidad, hecho en tiempo y forma. Al grano y sin sorpresas. Orientado al negocio.

Prácticas

Test Driven Development

Pair Programming

Entrega continua

Desde que conocimos NCrunch y Resharper no queremos programar sin ninguna de estas dos herramientas. Todos los refactors los hacemos automáticamente con Resharper para estar seguros en todo momento que no se rompe nada.

Después de cada "push" al control de versiones estamos tranquilos de poder subir a producción si hiciera falta. Aunque hemos usado TFS, preferimos sistemas que admitan commits a nivel local antes de compartir los cambios con los compañeros, por lo que habitualmente funcionamos con Git o Mercurial.

En el frontend por supuesto hacemos también TDD usando para ello Jasmine, o a veces Mocha. Chai.js para los asserts y Sinon para los mocks.

Testing

Unitario, Integración y E2E

Selenium WebDriver

Behavior Driven Development

Por supuesto todas nuestras features la dirigimos mediante tests (TDD) y también testeamos la arquitectura y la infraestructura. NUnit y FluentAssertions son nuestra combinación más habitual, acompañado de Moq o NSubstitute para los mocks. En algunos proyectos usamos SpecFlow (el Cucumber de .Net) como herramienta de comunicación en la que volcar los requisitos.
